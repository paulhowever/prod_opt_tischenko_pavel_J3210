# Анализ результатов экспериментов по методам оптимизации

## 1. Краткий вывод

Для **вещественных чисел (`f64`)** результаты очень хорошие:

- **Newton Method** оказался самым сильным на всех трёх задачах:
  - дошёл до минимума за **1 итерацию**,
  - дал точное или практически точное решение,
  - использовал минимум вызовов функции и градиента.
- **Gradient Descent** тоже работает хорошо на `Quad6WellCond` и `Rosenbrock3`, но сильно медленнее Ньютона.
- На **`Quad4IllCond`** градиентный спуск показал слабость: он **упёрся в лимит 10000 итераций** и не дошёл до минимума.
- **Nelder–Mead** на вещественных числах нашёл хорошие решения, но почти везде **очень дорогой по числу вызовов функции** и тоже часто упирается в лимит итераций.

Для **конструктивных чисел** картина намного хуже:

- интервалы часто остаются **слишком широкими**;
- решения нередко выглядят **численно нестабильными**;
- особенно плохо ведёт себя **Gradient Descent** на квадратичных задачах;
- **Nelder–Mead** работает устойчивее, но точность всё равно слабая;
- на текущей реализации конструктивные числа пока выглядят скорее как **демонстрация подхода**, а не как надёжный вычислительный режим.

## 2. Детальный анализ по `f64`

## Quad6WellCond

### Gradient Descent
- `best value = 0`
- решение очень близко к нулю по всем координатам
- **991 итерация**, `992` вызова функции и градиента

**Вывод:** метод сошёлся корректно, но довольно медленно.

### Nelder–Mead
- `best value = 0`
- решение тоже фактически нулевое
- **10000 итераций**, `38350` вызовов функции

**Вывод:** минимум найден, но цена очень высокая. Для гладкой квадратичной функции это невыгодно.

### Newton Method
- `best value = 0`
- `best x = [0, 0, 0, 0, 0, 0]`
- **1 итерация**, `2` вызова функции, `2` вызова градиента

**Вывод:** лучший метод на этой задаче. Это ожидаемо: квадратичная хорошо обусловленная функция идеально подходит для Ньютона.

## Quad4IllCond

### Gradient Descent
- `best value = 100`
- одна координата осталась равной `1.0`
- **10000 итераций**, достигнут лимит

**Вывод:** явная проблема со сходимостью.  
Это классический эффект **плохой обусловленности**: шаг `0.01` для этой задачи неудачен, и метод плохо продвигается вдоль “трудного” направления.

### Nelder–Mead
- `best value = 0`
- координаты очень близки к нулю
- **10000 итераций**, `46104` вызова функции

**Вывод:** решение найдено лучше, чем у градиентного спуска, но вычислительно это дорого.

### Newton Method
- `best value = 0`
- `best x = [0, 0, 0, 0]`
- **1 итерация**

**Вывод:** снова лучший метод. На квадратичной задаче с доступной гессианной он практически идеален даже при плохой обусловленности.

---

## Rosenbrock3

### Gradient Descent
- `best value = 0`
- `best x ≈ [1, 1, 0]`
- **947 итераций**

**Вывод:** очень хороший результат. Для функции Розенброка это уже достойно, потому что поверхность обычно “овражная” и трудная для обычного градиентного спуска.

### Nelder–Mead
- `best value = 0`
- решение близко к `[1, 1, 0]`
- **10000 итераций**, `49060` вызовов функции

**Вывод:** тоже пришёл к хорошему ответу, но снова слишком дорогой.

### Newton Method
- `best value = 0`
- `best x = [1, 1, 0]`
- **1 итерация**

**Вывод:** по результатам выглядит идеально.  
Но здесь стоит быть аккуратным при интерпретации: если метод Ньютона на Розенброке стабильно приходит за 1 шаг из старта `[1,1,1]`, это может означать:
- либо функция выбрана так, что старт уже очень удачный,
- либо гессиан/градиент устроены особенно просто,
- либо в коде критерий останова слишком мягкий.

Это не обязательно ошибка, но это стоит проверить отдельно.

## 3. Детальный анализ по конструктивным числам

## Общая картина

Здесь главный показатель — не только midpoint, но и:

- **width** интервала,
- **eps**,
- устойчивость координат.

Если интервал огромный, midpoint сам по себе почти ничего не значит.

---

## Quad6WellCond

### Gradient Descent
- `best value` имеет гигантскую ширину: порядка `9.53e11`
- координаты тоже имеют ширину порядка `5.6e5`

**Вывод:** результат практически непригоден.  
Хотя midpoint около нуля, неопределённость слишком большая. Это признак сильного накопления интервальной ошибки.

### Nelder–Mead
- `best value` около `5.81`
- ширина интервала примерно `2.25`
- координаты около `0.88 ... 1.16`

**Вывод:** лучше по устойчивости, чем конструктивный Gradient Descent, но до правильного минимума далеко. Для квадратичной функции это плохой результат.

**Вывод:** фактически развал метода.  
Именно здесь плохая обусловленность вместе с интервальной арифметикой даёт сильнейшую потерю точности.

### Nelder–Mead
- `best value` около `40.78`
- ширина очень большая
- координаты далеки от нуля

## 4. Сравнение методов

## По качеству решения на `f64`
1. **Newton Method** — лучший
2. **Gradient Descent** — хороший, но медленнее
3. **Nelder–Mead** — рабочий, но дорогой

## По устойчивости на конструктивных числах
1. **Nelder–Mead** — местами чуть стабильнее
2. **Gradient Descent** — часто сильно раздувает интервалы

Но в целом ни один метод в constructive-режиме пока не даёт действительно сильного качества.

---

## 5. Основные причины такой картины

### Почему `f64` всё хорошо
- квадратичные функции удобны для Ньютона;
- обычная вещественная арифметика не раздувает интервалы;
- Rosenbrock3 из данного старта тоже оказался решаемым.

### Почему constructive-режим плохой
Скорее всего, проблема в одном или нескольких пунктах:

- слишком грубая интервальная арифметика;
- быстрое накопление ошибки при множестве операций;
- слишком большой шаг у Gradient Descent;
- отсутствие специальных техник для interval / constructive optimization;
- критерий останова не учитывает ширину интервала;
- Nelder–Mead не очень хорошо согласован с интервальным сравнением значений.

## 6. Практические выводы для отчёта

Можно написать так:

> Эксперименты показали, что на вещественных числах метод Ньютона является наиболее эффективным на рассматриваемых тестовых функциях: он достигает минимума за минимальное число итераций и вызовов функции. Градиентный спуск показывает хорошую сходимость на хорошо обусловленных задачах и на функции Розенброка, однако существенно проигрывает на плохо обусловленной квадратичной функции. Метод Нелдера–Мида способен находить хорошие решения, но требует значительно большего числа вызовов функции.

> Для конструктивных чисел результаты существенно хуже: интервалы часто остаются широкими, а на плохо обусловленных задачах наблюдается резкое накопление погрешности. Это показывает, что прямая адаптация классических методов оптимизации к конструктивной или интервальной арифметике требует дополнительных механизмов стабилизации.

## 7. Что улучшить дальше

### Для вещественного режима
- добавить итоговую таблицу сравнения методов;
- считать норму градиента в конце;
- выводить причину остановки: `converged` / `max_iters` / `stagnation`.

### Для constructive-режима
- уменьшить шаг Gradient Descent;
- добавить backtracking line search;
- использовать критерий останова не только по значению, но и по ширине интервала;
- печатать отдельно midpoint и width;
- попробовать preconditioning для плохо обусловленной квадратичной задачи;
- ограничивать разрастание интервалов или менять модель constructive number.
